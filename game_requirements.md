# 🫧 ゲルぴよバウンス — ゲーム要件定義書

> **バージョン**: v1.0（初版）  
> **最終更新**: 2026-06-15  
> **ジャンル**: バウンスアクション（Doodle Jump風 × ゲル弾力メカニクス）  
> **プラットフォーム**: スマホブラウザ（縦画面、PWA対応可）

## 1. ゲーム概要

### 1.1 コンセプト

ゲルぴよ（ゲル素材のヒヨコキャラ）が、ぷるぷる弾む足場を跳び上がっていくバウンスアクションゲーム。  
タップで高くジャンプ、長押しで滞空。足場の種類によって跳ね方が変わる。  
**「簡単に遊べて、気づいたらもう1回やってる」** を目指す。

### 1.2 ゲームの流れ（1プレイ）

```
タイトル画面 → タップでスタート → 上昇しながら足場を跳ぶ
→ 足場を踏み外す or 画面下に落下 → ゲームオーバー
→ スコア表示 → タップで即リスタート
```

### 1.3 プレイ時間

- 1プレイ: **30秒〜2分**（上級者ほど長く生存）
- セッション: **5〜15分**（「あと1回」ループ）

## 2. コアメカニクス

### 2.1 基本操作

| 操作 | アクション | 詳細 |
|------|-----------|------|
| **タップ（タッチダウン）** | バウンス（ジャンプ） | 足場に着地中にタッチした瞬間にバウンス発動 |
| **長押し（空中でタッチ保持）** | 滞空（ホバリング） | 空中でタッチ保持するとゲルが膨らんでゆっくり降下（最大1.0秒） |
| **チルト（傾き）** | 横移動 | 端末の傾きで左右移動 |
| **キーボード（PC）** | 横移動 + ジャンプ | 左右矢印キーで移動、スペースキーでバウンス＆滞空（開発・テスト用） |

> **操作方針**: 片手（親指のみ）で完結する操作体系とする。  
> 横移動はチルト（加速度センサー）専用。タップ＝バウンスとの衝突を避けるため、左右タップでの横移動は非対応。  
> **操作フロー**: タッチダウン → バウンス発動 → 指を離さずに保持 → 滞空モード → 離す → 通常落下  
> **滞空チャージ**: 足場に着地すると滞空ゲージが全回復。空中で使い切ると通常落下に戻る。

### 2.2 物理挙動

#### 重力と跳ね

```
基本重力: 0.4 px/frame²
ジャンプ初速: -12 px/frame（上方向）
滞空時重力: 0.08 px/frame²（通常の1/5）
滞空最大時間: 1.0秒（キャラ能力で延長可能、最大1.5秒）
滞空チャージ: 足場着地で全回復
横移動速度: ±4 px/frame
画面端処理: ワープ（左端→右端、右端→左端）
```

#### ゲルぴよの変形アニメーション

- **着地時**: 縦方向に潰れて横に広がる（squash）→ 元に戻る（0.15秒）
- **ジャンプ時**: 縦方向に伸びる（stretch）→ 元に戻る（0.1秒）
- **滞空時**: 全体がぷるぷる揺れる（サイン波で振幅2px、周期0.3秒）
- **落下中**: やや縦長に伸びる

### 2.3 足場（プラットフォーム）の種類

| 種類 | 見た目 | 挙動 | 出現頻度 |
|------|--------|------|---------|
| **ノーマルゲル** | 半透明の緑ゲル | 通常のバウンス | 60% |
| **スーパーゲル** | 光る金色ゲル | 2倍の高さでバウンス | 10% |
| **ふわふわゲル** | ピンクの柔らかゲル | バウンス高度は低いが、次の落下がゆっくり（3秒間重力50%） | 10% |
| **壊れるゲル** | ひび割れたゲル | 1回踏むと崩壊（着地後0.3秒で消滅） | 15% |
| **移動ゲル** | 左右に揺れるゲル | 水平に往復移動（速度2px/frame） | 5% |

> 足場の出現ルール:  
> - 画面上部に向かってランダム配置  
> - 隣り合う足場間の縦距離: **60〜120px**（ジャンプで届く範囲）  
> - 横位置: ランダム（ただし極端な偏りを防ぐため3分割ゾーンでバランス）  
> - 高度が上がるほど「壊れるゲル」「移動ゲル」の割合が増加

### 2.4 スコアシステム

```
スコア = 到達した最大高度（ピクセル単位 → 10で割って表示）
ボーナス:
  - コイン取得: +10点/枚
  - スーパーゲル着地: +5点
  - 連続バウンス（足場を踏み外さずに連続着地）: 3連続でコンボ開始（×1.5, ×2, ×3... 上限×5）
  - 踏み外し or ゲームオーバーでコンボリセット
```

## 4. ゲームオーバー条件

- ゲルぴよが**画面下端より下**に落下した場合
- 画面はゲルぴよの最高到達点に追従してスクロールし、**下方向には戻らない**

## 8. サウンド（Web Audio API合成）

すべての効果音はWeb Audio APIのオシレーターで合成し、外部ファイルに依存しない。

| イベント | 音色 | 仕様 |
|---------|------|------|
| **バウンス（通常）** | ぽよん | サイン波 300Hz→600Hz、0.1秒、軽いリバーブ |
| **バウンス（スーパー）** | ぴょいーん！ | サイン波 400Hz→1200Hz、0.2秒、強いリバーブ |
| **コイン取得** | チリン | 矩形波 800Hz→1200Hz→800Hz、0.08秒 |
| **滞空開始** | ふわぁ | サイン波 200Hz、ボリューム徐々に上昇、ビブラート |
| **壊れるゲル** | パリッ | ノイズ波 0.05秒 |
| **ゲームオーバー** | ぐにゃ〜 | サイン波 600Hz→100Hz、0.5秒、ピッチベンド |
| **ハイスコア更新** | ファンファーレ | 矩形波 C5→E5→G5→C6、各0.15秒 |

### 8.1 音声制御

- 画面右上にミュート切替ボタン（🔊 / 🔇）
- 設定はlocalStorageに保存
- 初回アクセス時はミュート（ブラウザ自動再生ポリシー対応）→ 初回タップで解除

### 9.3 ゲームループ

```javascript
// 60FPS固定（requestAnimationFrame + デルタタイム補正）
function gameLoop(timestamp) {
    const deltaTime = timestamp - lastTimestamp;
    lastTimestamp = timestamp;
    
    update(deltaTime);  // 物理演算・衝突判定・状態更新
    render();           // Canvas描画
    
    requestAnimationFrame(gameLoop);
}
```

### 9.4 画面サイズ・レスポンシブ対応

```
論理解像度: 390 × 844 px（iPhone基準 9:19.5比率）
実デバイス: viewport全体にCanvasをフィット（アスペクト比維持）
スケーリング: CSS transform scale でレターボックス対応
```

- `window.resize` イベントでリアルタイムリサイズ
- Canvas内の座標はすべて論理解像度で計算

### 9.5 パフォーマンス目標

- 60FPS安定（中〜低スペックスマホ含む）
- 画面外のオブジェクトは描画・更新をスキップ
- パーティクルはオブジェクトプールで管理（GC抑制）

## 13. チュートリアル

- 初回〜3回目のプレイ時に操作ヒントを表示
- 第1プレイ: 「タップでジャンプ！」（最初の足場で表示）
- 第2プレイ: 「長押しで浮けるよ！」（空中で表示）
- 第3プレイ: 「スマホを傾けて移動！」
- ヒントは画面中央に吹き出しで表示、2秒後にフェードアウト
- 表示済みフラグはlocalStorageに保存

## 14. 一時停止画面

- 右上の一時停止ボタン（⏸）タップで発動
- ゲームループを停止（requestAnimationFrame を中断）
- ゲーム画面を暗転（50%透過の黒オーバーレイ）
- 中央に「PAUSE」テキスト
- ボタン: 「つづける」「やめる（タイトルへ）」「音声ON/OFF」

## 15. 難易度曲線

| 高度帯 | ノーマル | スーパー | ふわふわ | 壊れる | 移動 | 特記 |
|--------|---------|---------|---------|-------|------|------|
| 0〜1000 | 80% | 10% | 10% | 0% | 0% | チュートリアルゾーン |
| 1000〜3000 | 60% | 10% | 10% | 15% | 5% | 基本ゾーン |
| 3000〜5000 | 45% | 10% | 5% | 25% | 15% | 中級ゾーン |
| 5000〜8000 | 30% | 10% | 5% | 30% | 25% | 上級ゾーン |
| 8000〜 | 20% | 5% | 5% | 35% | 35% | 地獄ゾーン |

- 足場間距離も高度に応じて拡大: 基本60〜100px → 高度8000以降は80〜140px
- 移動ゲルの速度も高度に応じて加速: 基本2px/frame → 高度8000以降は3.5px/frame
