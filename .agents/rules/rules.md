# ワークスペース開発ルール（Web & ゲームアプリ特化）

このワークスペースでは、スマートフォンブラウザで軽快かつ美しく動作するアプリケーション（特にゲームやインタラクティブなWebアプリ）を開発します。

## 1. テクノロジースタック & 実装方針
- **Core**: HTML5, Vanilla CSS, Vanilla JavaScript (ES6) を原則とする。フレームワーク（React/Next.js/Vite等）は上杉さんの明示的指示がある場合のみ使用する。
- **ゲーム開発**: 描画負荷を最小限に抑え、豊かなグラフィックを描くため、HTML5 Canvas API を用いたゲームループ（`requestAnimationFrame`）による実装を優先する。
- **レスポンシブデザイン**: スマートフォン（縦・横）およびPCの双方で、レイアウトが崩れずアスペクト比（例: 9：19.5）が維持されるコンテナを設計する。

## 2. ビジュアル & デザイン（Rich Aesthetics）
- **プレミアムテーマ**: 単色（赤・青など）のプレースホルダーを避け、ネオン調やグラデーションを用いたダークモード、美しく調和されたカラーパレット（HSL）を最優先で適用する。
- **アセット生成と透過処理**: 
  - 画像が必要な場合は画像生成ツールで作成し、プレースホルダーは使用しない。
  - 画像アセットに不要な背景色（黒背景など）が含まれる場合は、JavaScriptのロード処理時にオフスクリーンCanvas等を利用して自動透過（クロマキー透過）させるコードを実装し、製品クオリティを保証する。
- **アニメーション**: CSSアニメーションやCanvasの物理挙動を用い、ホバー、クリック、登場、退場時の滑らかなアニメーションを実装する。

## 3. 音声・オーディオ（シンセサイザー合成）
- **外部アセット依存の排除**: 効果音が必要な場合、サーバーへの負担やロード遅延を避けるため、Web Audio API を活用したオシレーター（発振器）による効果音（ジャンプ音、取得音等）のリアルタイム合成発音を実装する。
- **オーディオ管理**: 必ずミュート（音声ON/OFF）切り替え機能をUIに設ける。

## 4. パフォーマンス基準（スマホブラウザ最適化）
- **フレームレート目標**: 60fps を維持する。処理落ちが発生する場合は描画対象の削減やロジックの簡略化で対応し、フレームスキップは最終手段とする。
- **オブジェクトプーリング**: パーティクル・弾・足場など頻繁に生成/破棄されるオブジェクトは、オブジェクトプールパターンを用いてGC（ガベージコレクション）停止を最小化する。
- **メモリ管理**: ゲームループ内での `new` によるオブジェクト生成を極力避け、事前確保した配列・オブジェクトを再利用する。
- **描画最適化**: Canvas の `drawImage` 呼び出し回数を最小限にまとめ、不要な再描画を避ける。オフスクリーンCanvasによる事前レンダリングを活用する。

## 5. タッチ入力の最適化（モバイルゲーム必須）
- **スクロール干渉の防止**: ゲームCanvas要素には `touch-action: none` を CSS で指定し、ブラウザのデフォルトスクロール・ズーム動作を無効化する。
- **イベントリスナー設定**: タッチイベント（`touchstart`, `touchmove`, `touchend`）は `{ passive: false }` で登録し、`preventDefault()` でブラウザのデフォルト動作を確実に抑制する。
- **入力遅延の排除**: `<meta name="viewport">` に `user-scalable=no` を含め、ダブルタップズームによる300ms遅延を排除する。
- **マルチタッチ考慮**: 意図しないマルチタッチによる誤動作を防ぐため、最初のタッチポイントのみを処理対象とするガード処理を実装する。

## 6. ローカルストレージ管理（データ永続化）
- **保存先**: `localStorage` を使用し、ゲームデータ（ハイスコア、アンロック状況、設定）を永続化する。
- **データ構造**: 保存データはバージョン番号付きの単一JSONオブジェクトとして管理する（例: `{ version: 1, highScore: 0, unlockedCharacters: [], settings: {} }`）。
- **マイグレーション**: データ構造を変更する際は、バージョン番号を更新し、旧バージョンからの自動マイグレーション関数を実装する。古いデータが消失しないことを保証する。
- **エラーハンドリング**: `localStorage` が利用不可（プライベートブラウジング等）の場合は、メモリ内フォールバックで動作を継続する。データ消失の警告をユーザーに表示する。

## 7. PWA対応方針
- **manifest.json**: アプリ名、アイコン（複数サイズ）、テーマカラー、`display: standalone`、`orientation: portrait` を設定する。
- **Service Worker**: オフラインでもプレイ可能にするため、ゲームアセット（HTML/CSS/JS/画像）をキャッシュするService Workerを実装する。Cache-First戦略を基本とする。
- **インストールプロンプト**: `beforeinstallprompt` イベントを捕捉し、適切なタイミング（例: 3回目のプレイ後）でホーム画面追加を促すUIを表示する。
- **ステータスバー対応**: PWAモードでのセーフエリア（ノッチ・ステータスバー）を `env(safe-area-inset-*)` で考慮し、UIが隠れないようにする。

## 8. ファイル構成規約
- プロジェクトルートに以下のディレクトリ構成を採用する:
  ```
  /
  ├── index.html          # エントリーポイント
  ├── style.css           # グローバルスタイル
  ├── manifest.json       # PWA マニフェスト
  ├── sw.js               # Service Worker
  ├── src/                # JavaScriptソース
  │   ├── main.js         # エントリー・初期化
  │   ├── game.js         # ゲームループ・コアロジック
  │   ├── input.js        # タッチ入力管理
  │   ├── audio.js        # Web Audio API 音声合成
  │   ├── storage.js      # ローカルストレージ管理
  │   └── ui.js           # UI（メニュー・スコア表示等）
  ├── assets/             # 画像・スプライト素材
  │   ├── characters/     # キャラクター画像
  │   └── ui/             # UI用アイコン・背景
  └── 素材/               # 元素材（開発用・直接参照しない）
  ```
- **命名規則**: ファイル名・変数名はキャメルケース（`gameLoop`, `playerScore`）、定数は大文字スネークケース（`MAX_SPEED`, `GRAVITY`）を使用する。
- **モジュール管理**: ES6の `import/export` を使用し、各ファイルの責務を明確に分離する。循環参照を避ける。
